# Master Skill Tree

> Every single skill/topic that exists between mid-level dev and CTO, categorized.

## Category A: Technical Skills (Hard Skills)

```
A1. CORE ENGINEERING DEPTH
    A1.1  Advanced programming (design patterns, SOLID, DRY, KISS)
    A1.2  Data structures & algorithms (beyond interview prep — real usage)
    A1.3  Testing strategies (unit, integration, e2e, contract, chaos)
    A1.4  Code architecture (clean architecture, hexagonal, DDD)
    A1.5  Performance optimization (profiling, memory, CPU, I/O)
    A1.6  Debugging complex systems (distributed tracing, log analysis)
    A1.7  Security fundamentals (OWASP, auth patterns, encryption)
    A1.8  API design (REST, GraphQL, gRPC, versioning, pagination)
    A1.9  Database mastery (indexing, query plans, sharding, replication)
    A1.10 Concurrency & parallelism (async, threads, race conditions)

A2. SYSTEM DESIGN & ARCHITECTURE
    A2.1  System design fundamentals (load balancers, caches, queues)
    A2.2  Distributed systems (CAP theorem, consistency, partitioning)
    A2.3  Microservices architecture (service boundaries, communication)
    A2.4  Event-driven architecture (event sourcing, CQRS, Kafka)
    A2.5  Scalability patterns (horizontal/vertical, database scaling)
    A2.6  Reliability engineering (redundancy, failover, circuit breakers)
    A2.7  Data architecture (data lakes, warehouses, ETL, streaming)
    A2.8  Architecture decision records (ADRs) and trade-off analysis
    A2.9  Migration strategies (monolith→microservices, cloud migration)
    A2.10 Multi-region / global architecture

A3. CLOUD & INFRASTRUCTURE
    A3.1  Cloud platforms (AWS / GCP / Azure — core services)
    A3.2  Infrastructure as Code (Terraform, Pulumi, CloudFormation)
    A3.3  Containers (Docker) and orchestration (Kubernetes)
    A3.4  CI/CD pipelines (GitHub Actions, Jenkins, ArgoCD)
    A3.5  Monitoring & observability (metrics, logs, traces — Datadog, Grafana)
    A3.6  Cost optimization (cloud spend management, FinOps basics)
    A3.7  Networking (DNS, CDN, VPC, load balancing, TLS/SSL)
    A3.8  Serverless architecture (Lambda, Edge Functions)

A4. AI & EMERGING TECH (Critical for 2025+)
    A4.1  AI/LLM integration (OpenAI, Anthropic APIs, RAG, vector DBs)
    A4.2  Prompt engineering and AI product design
    A4.3  AI tool mastery (Cursor, Copilot, Claude — 10x your output)
    A4.4  Understanding ML basics (enough to lead AI teams)
    A4.5  AI strategy (build vs buy, cost analysis, when AI fits)
```

## Key Decisions at This Phase

```
DECISION POINT: IC Track or Management Track?

Choose IC (Staff → Principal → Distinguished) if:
  ✓ You love solving hard technical problems
  ✓ You don't enjoy meetings and 1-on-1s
  ✓ You want to code 60-80% of your time
  ✓ You get energy from building, not from people

Choose Management (Tech Lead → EM → Director → VP → CTO) if:
  ✓ You care more about team success than personal code
  ✓ You enjoy mentoring and growing people
  ✓ You're good at communication and politics
  ✓ You get energy from people, not just problems
```

## You're Ready for Phase 3 When...

```
✅ You can design systems that handle millions of users
✅ You've written ADRs/RFCs that influenced your team's direction
✅ Other engineers (not just juniors) seek your technical opinion
✅ You've mentored someone from junior to mid-level
✅ You've led a cross-team project or initiative
✅ You can present a technical strategy to non-engineers clearly
✅ You've made the IC vs Management track decision
```

# IC Track vs Management Track

> Complete comparison of the two paths from Staff level onward.

| Dimension | IC Track | Management Track |
|---|---|---|
| **What you do daily** | Design systems, write critical code, solve hardest problems, write RFCs | 1-on-1s, hiring, team planning, cross-team coordination, remove blockers |
| **Code time** | 50-70% (Staff), 30-50% (Principal) | 20-30% (EM), 0-10% (Director+) |
| **Meeting time** | 20-30% | 50-70% |
| **How you're measured** | Technical impact, systems designed, problems solved | Team output, retention, hiring, shipping velocity |
| **Hardest part** | Staying relevant technically while doing more leadership | Letting go of code, handling people problems |
| **Salary ceiling** | $400-600K+ / ₹1-1.5Cr (Distinguished) | $400K-1.2M+ / ₹1-3Cr+ (CTO) |
| **Job security** | Very high (hard to replace) | High but political (reorgs can eliminate roles) |
| **Switching cost** | Can switch to management later (common) | Hard to switch back to IC after 3+ years |
| **Personality fit** | Introverted OK, loves depth | Must enjoy people, comfortable with ambiguity |
| **Path to CTO** | Possible but uncommon (usually via Architect → CTO at startup) | Natural path (Director → VP → CTO) |
| **Skills that matter most** | A1-A4 (technical) + B1-B2 (communication + influence) | B2-B3 (leadership + people) + C1-C3 (business + strategy) |
